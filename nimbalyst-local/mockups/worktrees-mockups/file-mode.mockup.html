Authentication System

Overview

This document outlines the OAuth-based authentication system with support for Google and GitHub providers. The system uses JWT tokens for session management and includes automatic token refresh.

Configuration

Set up your OAuth credentials in the auth.config.ts file:

export const authConfig = {
  google: {
    clientId: process.env.GOOGLE_CLIENT_ID,
    clientSecret: process.env.GOOGLE_CLIENT_SECRET,
    redirectUri: 'http://localhost:3000/auth/google/callback'
  },
  github: {
    clientId: process.env.GITHUB_CLIENT_ID,
    clientSecret: process.env.GITHUB_CLIENT_SECRET,
    redirectUri: 'http://localhost:3000/auth/github/callback'
  }
};

Session Management

Sessions are managed using JWT tokens with a 7-day expiration. Refresh tokens are stored securely and can be used to obtain new access tokens without requiring re-authentication.

Implementation Details

The authentication flow consists of three main components:

1. OAuth provider integration (oauth.ts)

2. Session middleware (session.ts)

3. Token management and validation
